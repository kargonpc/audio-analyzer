# 🎵 Smart Audio Analyzer (DSP Edition)

Высокопроизводительная CLI-утилита для автоматического анализа музыкальных аудиофайлов (BPM и Тональность). Проект создан для организации библиотек сэмплов и битов.

Вместо использования тяжеловесных Machine Learning моделей, анализатор опирается на **классическую цифровую обработку сигналов (DSP)** и продвинутые математические алгоритмы библиотеки `librosa`.

## 🧠 Архитектура и Математика (DSP vs ML)

Проект использует алгоритм **Крамхансля-Шмуклера** (Krumhansl-Schmuckler) для профилирования тональности и алгоритм поиска пиков для определения темпа. 

В процессе разработки и тестирования на современных жанрах (Rage, Trap, Alt-Rock) были выявлены физические ограничения математического анализа по сравнению с коммерческими ML-решениями:

1. **Проблема 808-х басов (Сатурация):** Искаженные басовые партии генерируют мощные обертоны (особенно квинты). Математика (STFT) считывает обертон как отдельную ноту, из-за чего может перепутать, например, `Bb Major` и `F Major`.
   * **Решение:** Внедрен программный **Bandpass-фильтр (60Hz - 2000Hz)** (через `scipy.signal.butter`), срезающий саб-бас и верхние частоты (перкуссию), оставляя чистую гармоническую середину.
2. **Проблема шума и перкуссии:**
   * **Решение:** Использование алгоритма **HPSS (Harmonic-Percussive Source Separation)** перед анализом тональности.
   * Переход с классического STFT на логарифмическое **CQT (Constant-Q Transform)**, которое точнее распределяет частоты по музыкальным нотам.
3. **Плавающий темп и параллельные тональности:**
   * **Решение:** Алгоритм собирает гистограмму локальных темпов по всему треку и выдает **Топ-2 вероятностей** для BPM и тональности (включая расчет параллельной гармонии по кварто-квинтовому кругу).

## 🚀 Особенности

* **Мультипроцессинг:** Использование `ProcessPoolExecutor`. Скрипт анализирует аудиофайлы параллельно, загружая все ядра CPU (ускорение в 8-10 раз).
* **Оптимизация I/O:** Для анализа подгружаются только 30 секунд из середины аудиофайла, что радикально ускоряет работу с длинными треками.
* **Экспорт:** Возможность генерации `library_index.json` (локальной базы данных сэмплов).

## 💻 Установка и запуск

1. Клонируйте репозиторий и установите зависимости:
```bash
pip install -r requirements.txt
```
2. Запуск в безопасном режиме (только анализ):
```bash
python organizer.py /путь/к/папке/с/сэмплами
```
3. Запуск с физическим переименованием файлов:
```bash
python organizer.py /путь/к/папке --rename
```
4. Экспорт результатов в JSON базу данных:
```bash
python organizer.py /путь/к/папке --export-json
```
